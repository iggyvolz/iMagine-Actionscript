Download

This application is written in ActionScript. Therefore it must be compiled before it can be run. As we are not at a stable release point, the quickest and surest way to do it is to compile it yourself. To do this, enter the following in your terminal (assuming Unix or Mac):

  1. cd ~
  2. git clone https://github.com/iggyvolz/iMagine ## Clone iMagine
  3. cd iMagine ## Move into iMagine
  4. git submodule update --init --recursive ## Clone ASUnit
  5. cd ~
  6. wget http://download.macromedia.com/pub/flex/sdk/flex_sdk_4.6.zip ## Download the Flex SDK
  7. unzip flex_sdk_4.6.zip ## Unzip the Flex SDK
  8. echo "export PATH=$PATH:$HOME/flex_sdk/bin">>~/.profile ## Add the Flex SDK Bin to your $PATH
  9. source ~/.profile ## Re-run ~/.profile so you don't have to restart terminal
  10. cd iMagine ## Move into iMagine

Now you can just run make build debug and open up iMagine! Note that in certain versions of OS X you may need to follow the instructions here to use the 64-bit Java player instead of the 32-bit player.

If you would prefer not to compile yourself, the latest successfully compiling binary is pushed here about 1 minute after each commit.

If you would prefer to simply use point-releases (these point-releases are tested more thoroughly), please see the Github Releases Page.

Syntax

All commands are case-insensitive and will be converted immediately to all lowercase. The syntax is: person.action(parameter). However, the person and/or the parameter can be omitted for some functions. For example, the function version requires no name or parameters. Any of these will do the same thing:

  • i.version()
  • i.version(foo)
  • i.version
  • version()
  • version(foo)
  • version

As you noticed in the example above, you can use the person i. This is the person you are currently playing as (you can switch using the switchto command). You can also refer to anyone by name in the person spot (for example, tony.reset).

Functions

Changeto

Note - Name is technically optional, but this function is pointless if the name is not included or "i" is used. Name must be a valid person. Parameters are optional and will be ignored if included.

This changes the value of the person i (or no person) to the person in the name.

Magine

Note - Name is optional, defaulting to "i". If included, it must be a valid person. Parameter is required to be a valid dream creature that is owned by the person listed in name and be in the dream plane before being magined. In addition, the person must have at least 100 energy to spend on magining the dream creature.

This "magines" a Dream Creature. It costs 100 energy (subject to change). It brings the Dream Creature in parameter from the Dream Plane (where they rest) to the real world, where they can fight (coming soon). For more info, see the Magi Nation Wiki. Person must be a valid person. Parameter must be a Dream Creature that is in the Dream Plane. Person must have Parameter's Animite (the stone used to magine dream creatures), that is, you cannot magine someone else's Dream Creature.

ToTheDreamPlane

Note - Name is required to be a valid dream creature that is in battle before this action is called. Parameter is optional and will be ignored if included.

This returns a Dream Creature to the Dream Plane. For more info, see the Magi Nation Wiki. Person must be a Dream Creature that is active (not in the Dream Plane). Parameter is ignored.

Version

Note - Name and parameters are optional and will be ignored if included.

This returns the current version of the software. If it is a development version, it will also include the latest commit number.
